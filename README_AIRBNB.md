# Сравнение линтера Devcom и Airbnb

## Что есть в текущем линтере

- Конфигурация строится на `@eslint/js` и `typescript-eslint` с включёнными рекомендованными типобезопасными правилами и настройкой `projectService` для корректной работы с `tsconfig` проекта.【F:configs/base.mjs†L1-L25】


- Встроена интеграция с Prettier через `eslint-plugin-prettier/recommended`, что превращает форматирование в ошибки ESLint и синхронизирует стиль с правилами из `prettier.js` (например, обязательные точки с запятой, одинарные кавычки, ширина строки 120).【F:configs/base.mjs†L1-L25】【F:prettier.js†L1-L6】


- Управление импортами: запрет дубликатов и расширений в путях, обязательная сортировка импортов с группировкой по внешним/внутренним модулям и абсолютным/относительным путям (simple-import-sort).【F:configs/rules/import.rule.mjs†L1-L19】【F:configs/rules/import-sort.rule.mjs†L1-L19】


- Базовые TypeScript-правила: явный тип возврата функций, единый стиль объявления типов через `interface`, запрет висящих промисов, строгие настройки для неиспользуемых переменных, требование указывать модификаторы доступа и порядок членов классов.【F:configs/rules/typescript.rule.mjs†L1-L39】


- Общие JS-правила: обязательные точки с запятой и запрет использования `Symbol` и `BigInt` в средах без нативной поддержки по рекомендациям Airbnb JavaScript Style Guide (Types 1.1–1.2).【F:configs/base.mjs†L7-L25】【F:configs/rules/javascript.rule.mjs†L1-L30】


- Набор игнорируемых путей (node_modules, сборочные артефакты, документация и т.д.) для ускорения линтинга.【F:configs/rules/ignore.rule.mjs†L1-L16】


- Дополнительный строгий профиль добавляет запрет `any`, правила именования переменных (camelCase/PascalCase), ограничение одного класса/интерфейса/типа/перечисления на файл и требование модульной структуры проекта.【F:configs/strict.mjs†L1-L20】【F:configs/rules/one-module.rule.mjs†L1-L38】

## Что есть в Airbnb JavaScript Style Guide

- Официальный пакет `eslint-config-airbnb` ориентирован на JavaScript и React: включает плагины `eslint-plugin-react`, `eslint-plugin-react-hooks`, `eslint-plugin-jsx-a11y` и `eslint-plugin-import` для строгих правил JSX, хуков и доступности.


- Предпочитает классический стиль JavaScript: обязательные точки с запятой, двойные кавычки в JSX-атрибутах, пробелы вокруг операторов, ограничение вложенности и длины строк, строгие правила для деструктуризации, `prefer-const`, `no-var` и контроль порядка методов в классах.


- Набор правил рассчитан на JavaScript/JSX: TypeScript поддерживается только через отдельные community-плагины и не входит в базовый пресет.


- Форматирование не завязано на Prettier: стиль определяется самими ESLint-правилами Airbnb, хотя в экосистеме есть адаптер `eslint-config-airbnb-prettier` для совместной работы.

## Что из Airbnb отсутствует в текущем линтере

- Нет React/JSX-специфичных проверок (`react`, `react-hooks`, `jsx-a11y`), поэтому качество компонентов и доступности остаётся на усмотрение проекта или требует отдельных плагинов.


- Семантика стиля отличается: текущий пресет использует Prettier и отдельные строгие правила (включая обязательные точки с запятой), но по-прежнему не покрывает многие детализированные требования Airbnb по деструктуризации, вложенности и структуре классов.


- Не покрываются многие JavaScript-ориентированные рекомендации Airbnb (строгая деструктуризация, предпочтение `const`, контроль вложенности и комплексные правила форматирования классов), потому что основной упор сделан на TypeScript-безопасность и сортировку импортов.


- Отсутствует встроенная поддержка доступности (a11y) и специальных правил для организации React-компонентов, которые входят в состав Airbnb-конфига.
