# Стайлгайд линтера

Документ описывает, какие рекомендации взяты из [Airbnb JavaScript Style Guide](https://leonidlebedev.github.io/javascript-airbnb) и какие правила расширены или добавлены в данном пресете.

## Наследуемые принципы Airbnb

- **Структура и форматирование кода.** Предпочтение `const`, избегание `var`, обязательные точки с запятой, пробелы вокруг операторов, аккуратная деструктуризация объектов и массивов.
- **Организация функций и классов.** Ограничение глубины вложенности, единообразный порядок методов в классах, явные значения по умолчанию и валидация аргументов.
- **Работа с массивами и объектами.** Использование `map`/`filter`/`reduce` вместо императивных циклов, отсутствие изменения аргументов функции, разворот массивов и объектов через спред.
- **React и JSX.** Проверки хуков, именование компонентов с заглавной буквы, обязательный `key` в списках, правила доступности (`jsx-a11y`) и согласованный стиль JSX-атрибутов.

## Дополнения поверх Airbnb

- Типобезопасные правила для TypeScript: включён `typescript-eslint` с рекомендованным набором, строгими проверками `any`, явными типами возвращаемых значений и контролем модификаторов доступа.
- Управление импортами через `eslint-plugin-simple-import-sort` и запрет дублирующих путей: единый порядок внешних, внутренних и относительных модулей.
- Интеграция с Prettier: форматирование выносится в ошибки ESLint, что позволяет синхронизировать стиль между JavaScript/TypeScript и React-кодом.
- Расширенный строгий профиль: ограничение одного публичного типа/класса/интерфейса/enum на файл и дополнительные правила именования для единообразия API.
